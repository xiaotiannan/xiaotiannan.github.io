如何在服务器上装lnmp且通过https解析到上面

服务器安装lnmp

提前准备

  • 一台电脑 —我的是mac系统
  • 一个域名—-在阿里云上,godaddy,namecheap 他们各个的不同,就不介绍了
  • 一台linux服务器–国内阿里云,我用的是vultr,可搭梯子,10美元6个月,最便宜的那种.

连接服务器

  • ssh root@110.110.110.110
  • 然后输入密码.

安装lnmp—linux+nginx+mysql+php

  • 一个个的配置,太烦了,就直接用网上的lnmp搭配lnmp一键安装包
  • 大约半小时你就安装好了.这个是lnmp伪静态管理
  • 这个是lnmp文件所在的位置lnmp所在的位置
  • 这个是简单的lnmp命令
  • 安装好以后通过ip是可以访问到服务器的.

我的配置

wget -c http://soft.vpser.net/lnmp/lnmp1.3-full.tar.gz && tar zxf lnmp1.3-full.tar.gz && cd lnmp1.3-full && ./install.sh lnmp

  • 输入服务器的密码
    之后都是跟这lnmp安装来

域名解析

  • 我的服务器ip是104.156.238.93
  • 域名在阿里云上这样解析就能通过域名访问服务器了.

通过https加密

  • 因为现在域名是通过默认的访问的需要加个虚拟主机

  • 之后一路按回车.

  • 到html目录中.cd /home/wwwroot/www.dong99.com/

  • 新建一个文件 里面写点东西然后访问www.dong99.com 是否出现写的东西vim index.html

  • 到home文件夹 cd /home

  • 新建文件夹ssl mkdir ssl

  • 下载Let’s encrypt 证书的脚本 到ssl 文件夹中

    • 1.wget https://raw.githubusercontent.com/xdtianyu/scripts/master/lets-encrypt/letsencrypt.conf
    • 2.wget https://raw.githubusercontent.com/xdtianyu/scripts/master/lets-encrypt/letsencrypt.sh
    • 3.更改执行权限chmod +x letsencrypt.sh
    • 查询文件有几个 ls 总共有2个文件 一个letsencrpt.sh 一个是letsencrypt.conf

  • 更改letsencrypt.conf文件 vim letsencrypt.conf
    更改内容,第一个是原内容,第二个是我修改后内容

  • 修改完之后运行./letsencrypt.sh letsencrypt.conf

    • 出现一个这样问题

    • 是英文在nginx 中是不能允许在浏览器上访问隐藏文件夹,所以需要去修改nginx的配置.



* 然后重新运行`./letsencrypt.sh letsencrypt.conf`

    <img src="/images/linux/06.png" class="blockquote-center"   style="display"/>
这个表示成功了.

* `ls` 看下文件有几个

<img src="/images/linux/07.png" class="blockquote-center"   style="display"/>
里面有9个文件,其中`www.crt-OLD-160713-150415`是可以删除了,这个是刚刚运行错误生成的文件.

修改nignx配置

  • 到vhost文件夹cd /usr/local/nginx/conf/vhost
  • 编辑文件 vim www.dong99.com.conf

  • 重定向到https 301

打完收工.但是letsencrypt 证书只能90天,我们需要写个定时的任务每月执行一次.

  1. 进入cron定时任务编辑crontab -e
  2. 输入0 0 1 * * /home/ssl/letsencrypt.sh /home/ssl/letsencrypt.conf >> /var/log/lets-encrypt.log 2>&1
    关于这个定时任务是每个月自动执行 letsencrypt.sh letsencrypt.conf 并且追加到log文件中,，2>&1，表示2的错误输出重定向等同于1.
  3. 查询这定时任务crontab -l
  4. 到ssl文件夹中编辑letsencrypt.sh加入每次更新完重启lnmp功能cd /home/ssl/letsencrypt.sh
  5. 到最后一行 将# 去掉,将最后一行改为lnmp restart.如果你是其他重启命名就改为其他.

每日身体锻炼总结

发表于 2016-06-09   |   分类于 body   |  

锻炼为了让自己的身体更好看点,为了让自己腹肌上的一坨肉变成2块腹肌,不要求变6块.

  • 已经坚持运动一周了,打算写下这篇文章,让自己时刻想着这件事情.并通过记录的方式思考和反思运动过程.

吃

  • 关于吃,由于现在在学校,不能像网上的教程一样,买东西自己做健康减肥增肌的食物吃,基本上都在学校食堂吃的
  • 早饭
    • 2个鸡蛋
    • 一杯豆浆
    • 一个玉米
    • 如果感觉还饿的话再来个菜包
    • 总花费大约5.5元
  • 中饭
    • 1两的米饭
    • 3个素材和1份牛肉土豆
    • 吃饭之前先喝1碗汤,然后在吃菜的时候,先把菜放在汤里将油去掉.学校食堂油特别多
    • 总花费大约9.2元

  • 晚饭

    • 1两的米饭
    • 一份包含鹌鹑蛋的素砂锅一份–大部分都是素材
    • 花费大约10元

      锻炼

  • 跑步

    • 跑步之前必须将身体活动开,保护自己的膝盖,一定要做运动前的准备活动.
    • 每天晚上大约9点开始跑步.跑12圈.跑完后衣服必定湿透.
    • 8点:30左右在居住的地方,在瑜伽垫上买了个瑜伽垫这个是我在淘宝上买的瑜伽垫大约20元每天做一边腹肌撕裂者,我是在好知上看到视频跟着视频做,腹肌撕裂者视频第一次做的时候特别难受,慢慢的也就习惯了.

睡

  • 每天大概12点睡,早上6:30起.中午睡个1小时的午觉.
  • 睡眠总时间大约7个半小时 到8小时之间

正则表达式高手攻略

发表于 2016-04-24   |   分类于 正则表达式   |  

正则表达式简单模式

    • 匹配开头是hello,结尾是world的,中间可以任意字符.
    • . 元字符匹配除了换行符以外的字符
    • * 代表着数量指的是它前面的内容连续重复使用任意次数
    • 匹配以字幕h开头的单词–先是\b单词的开头,接着是\w+至少一个的字母或者数字,最后是单词的结束\b
    • \w 匹配字母或者数字或者下划线或者汉字
    • +匹配前面的内容一次或者更多次数,与*不同的是*可以匹配0次
    • 这个类似于准确匹配的意思,
    • \.字符转义,如果你想查找字符串中的. + *之类的话就需要用\来进行转义查询,\\这个就是表示\本身
  4. [abc]+—abcabc
    • 只匹配由abc 三个字母组成得到至少一个字符串
    • [ ]匹配中括号中的字符串,例

正则表达式的正常模式

  1. \(?0\d{2}[) -]?\d{8}
    • 匹配几种格式的电话号码-(010)88888888 010-88888888 01088888888等
      • 分析:\(表示转义的(, ?表示它出现0次或者1次;之后接了一个0,后面跟着2个数字\d{2},然后是[) - ]?表示跟着)或-或空格中的一个出现1次或者0次,最后出现8个数字\d{8}.
    • ?表示前面的内容出现0次或者1次
    • 这个也能匹配到010)88888888 这样错误的格式要解决这样的问题需要用到分支条件

  2. 0\d{2}-\d{8}|0\d{3}-\d{7}–010-88888888 0817-8888888

    • 表示匹配到连字符分隔的电话号码
      • 一种是三位区号-010-88888888
      • 一种是四位区号0817-8888888
    • |是分支条件,指的是几种规则,如果满足任意一种斗应该完成匹配 有中文意思或的意思

  3. \(0\d{2}\)[- ]?\d{8}|0\d{2}[- ]?\d{8}—(010)-88888888 或010-8888888

    • \(转移(,接着是个0,然后匹配到2个数字,接着\)转移),然后匹配到-一次或者0次,最后接着8个数字

  4. \d{5}-\d{4}|\d{5}—-55555或88888-8888

    • 这个表达式用于匹配美国的邮政编码。美国邮编的规则是5位数字，或者用连字号间隔的9位数字。
    • 给出这个例子是因为它能说明一个问题：使用分枝条件时，要注意各个条件的顺序。如果你把它改成\d{5}|\d{5}-\d{4}的话，那么就只会匹配5位的邮编(以及9位邮编的前5位)。原因是匹配分枝条件时，将会从左到右地测试每个条件，如果满足了某个分枝的话，就不会去再管其它的条件了。

  5. ((2[0-4]\d|25[0-5]|[01]?\d\d?)\.){3}(2[0-4]\d|25[0-5]|[01]?\d\d?)——-255.255.255.255

    • 表示一个正确的IP地址
    • 因为ip地址中的每个数字都不能大于255,才写的这么多
    • 先看中括号里面的2[0-4]\d|25[0-5]|[01]?\d\d?
      • 先匹配一个2,然后加0-4中的一个,接着加数字一个—240
      • 或者匹配25,然后加[0-5]中的一个 —255
      • 先匹配[01]?表示0或者1一次或者0次,然后加一个\d数字,最后加个数字\d?匹配0次或者1次.
      • 这样就表示0-255都匹配到了.后面一个{3}表示重复3次,再后面的{}里面的内容是前面去掉.的内容,这样整个就表示0.0.0.0到255.255.255.255的所有IP地址了
