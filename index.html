JavaScript变量可以用来保存两种类型的值：基本类型值和引用类型值。基本类型的值源自以下5种基本数据类型：Undefined、Null、Boolean、Number和String。基本类型值和引用类型值具有以下特点：

基本类型值在内存中占据固定大小的空间，因此被保存在栈内存中；

从一个变量向另一个变量复制基本类型的值，会创建这个值的一个副本；

引用类型的值是对象，保存在堆内存中；

包含引用类型值的变量实际上包含的并不是对象本身，而是一个指向该对象的指针；

从一个变量向另一个变量复制引用类型的值，复制的其实是指针，因此两个变量最终都指向同一个对象；

确定一个值是哪种基本类型可以使用typeof操作符，而确定一个值是哪种引用类型可以使用instanceof操作符。

所有变量（包括基本类型和引用类型）都存在于一个执行环境（也称为作用域）当中，这个执行环境决定了变量的生命周期，以及哪一部分代码可以访问其中的变量。以下是关于执行环境的几点总结：

执行环境有全局执行环境（也称为全局环境）和函数执行环境之分；

每次进入一个新执行环境，都会创建一个用于搜索变量和函数的作用域链；

函数的局部环境不仅有权访问函数作用域中的变量，而且有权访问其包含（父）环境，乃至全局环境；

全局环境只能访问在全局环境中定义的变量和函数，而不能直接访问局部环境中的任何数据；

变量的执行环境有助于确定应该何时释放内存。

JavaScript是一门具有自动垃圾收集机制的编程语言，开发人员不必关心内存分配和回收问题。可以对JavaScript的垃圾收集例程作如下总结。

离开作用域的值将被自动标记为可以回收，因此将在垃圾收集期间被删除。

“标记清除”是目前主流的垃圾收集算法，这种算法的思想是给当前不使用的值加上标记，然后再回收其内存。

另一种垃圾收集算法是“引用计数”，这种算法的思想是跟踪记录所有值被引用的次数。JavaScript引擎目前都不再使用这种算法；但在IE中访问非原生JavaScript对象（如DOM元素）时，这种算法仍然可能会导致问题。

当代码中存在循环引用现象时，“引用计数”算法就会导致问题。

解除变量的引用不仅有助于消除循环引用现象，而且对垃圾收集也有好处。为了确保有效地回收内存，应该及时解除不再使用的全局对象、全局对象属性以及循环引用变量的引用。

如何阅读一本书笔记

发表于 2016-11-06   |   分类于 read book   |  

阅读的目标:

  • 为获得资讯而读:能增加资讯却不能增加理解力
  • 为求的理解而读:理解更多的事情
    • 开始时不相等的理解程度
    • 阅读的人一定要把不相等的理解力克服到一定程度内.
  • 阅读就是学习:指导型的学习,以及自我发现型学习之间的差异
    • 指导型学习称为”辅助的自我发现学习”
    • 非辅助型的自我学习-是阅读自我和世界的学习

  • 阅读的艺术包括了所有非辅助型自我发现学习的技巧

    • 敏锐的观察
    • 灵敏可靠的记忆
    • 想象的空间
    • 训练有素的分析

  • 强调阅读的重要性,将倾听作为第二.

    • 倾听是从一位出现在你面前学习的老师学习
    • 而阅读却跟一位缺席的老师学习.

  • 初学者的无知在于未学,而学者的无知在于学后

暗时间-笔记

发表于 2016-09-29   |   分类于 read book   |  

什么样的记忆和学习最有效

* 养成习惯,经常回顾一段时间所学到的东西.
* 创造回忆的机会
    * 与他人讨论.
    * 整理笔记,用来巩固淡化的记忆.
    * 书写-将一段时间学习的知识按照一个主题系统地“串”起来大大地丰富了知识之间的关联，平添无数提取线索.
  • 设身处地地“虚拟经历”别人经历过的事情,学习经验知识.
    • 实践的记忆比虚拟经历更加深刻,但是我们不能每次都实践.
  • 抽象和推广,举一反三.(归纳推理-inductive reasoning)
    • 联系比较自身的经历.
      *

好的学习习惯

  • Google&Wiki(不懂的事情,第一个要做的事情)
    • 看书的话,需要挑剔,不要拿到书就看,
    • 多做读书笔记,输入输出,一定要输出,(思考)
    • 关于思考,利用吃饭和睡觉的时候,弄一个问题在脑子里面,在思考中迷糊的睡着.
      • 思考还有一个好处就是,可以避免自己的焦虑.
  • 重要的事情一定要先做
  • 重要的事情要用大块的时间来完成,不要用零碎的时间,比如读书,不利于知识的组建.
  • 多看心里学的和思维的书籍,
  • 学习一项知识,先问自己三个重要的问题
    • 它的本质是什么?
    • 它的第一原则是什么
    • 它的知识结构是怎么样的?
  • 获得的多少并不是在于你学了多少或读了多少,而取决于你思考了多少和思考多深.
  • 善于利用小块时间.
