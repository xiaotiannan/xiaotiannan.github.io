暗时间-笔记

发表于 2016-09-29   |   分类于 read book   |     |  

什么样的记忆和学习最有效

* 养成习惯,经常回顾一段时间所学到的东西.
* 创造回忆的机会
    * 与他人讨论.
    * 整理笔记,用来巩固淡化的记忆.
    * 书写-将一段时间学习的知识按照一个主题系统地“串”起来大大地丰富了知识之间的关联，平添无数提取线索.
  • 设身处地地“虚拟经历”别人经历过的事情,学习经验知识.
    • 实践的记忆比虚拟经历更加深刻,但是我们不能每次都实践.
  • 抽象和推广,举一反三.(归纳推理-inductive reasoning)
    • 联系比较自身的经历.
      *

好的学习习惯

  • Google&Wiki(不懂的事情,第一个要做的事情)
    • 看书的话,需要挑剔,不要拿到书就看,
    • 多做读书笔记,输入输出,一定要输出,(思考)
    • 关于思考,利用吃饭和睡觉的时候,弄一个问题在脑子里面,在思考中迷糊的睡着.
      • 思考还有一个好处就是,可以避免自己的焦虑.
  • 重要的事情一定要先做
  • 重要的事情要用大块的时间来完成,不要用零碎的时间,比如读书,不利于知识的组建.
  • 多看心里学的和思维的书籍,
  • 学习一项知识,先问自己三个重要的问题
    • 它的本质是什么?
    • 它的第一原则是什么
    • 它的知识结构是怎么样的?
  • 获得的多少并不是在于你学了多少或读了多少,而取决于你思考了多少和思考多深.
  • 善于利用小块时间.

    时间管理

    • 在学习和思考中常问自己
      • 你的问题到底是什么?(提醒自己思考不要偏离问题)
      • 到现在为止,我到底有什么收获?(提醒自己时不时的去总结,整理学习的东西)
      • 设想需要讲给一个不懂的人听,(迫使自己去挖掘知识背后最本质,往往也是最简单的解释)
    • 重视知识的本质：对于程序员来说这一点尤其重要，程序员行业的知识芜杂海量，而且总是在增长变化。很多人感叹跟不上新技术。应对这个问题的办法只能是：抓住不变量。大量的新技术其实只是一层皮，背后的支撑技术其实都是十来年不变的东西。底层知识永远都不过时。算法数据结构永远都不过时。基本的程序设计理论永远都不过时。良好的编码习惯永远都不过时。分析问题和解决问题的能力永远都不过时。强大的学习能力和旺盛的求知欲永远都不过时。你大脑的思维方式永远都不过时。
    • 重视积累的强大力量,万事提前准备:计划定长点.5年计划.
    • 抬起头来: 时不时的抬起头来审视自己正在做的事情
      *

学习方法

  • 趁着对一件事情有热情的时候,一股脑的把万事开头难那个最难的阶段熬过去.
  • 根据主题来查阅资料,而不是根据资料查阅主题.
  • 在学习之前,先在大脑中积累充分的”疑惑感”,然后在去阅读.对问题的疑惑越大，在之前做的自己的思考越多，当看到解答之后印象就越深刻。
    • 有选择的阅读,技术性比较弱的或信息密集度比较低的部分快速阅读.
      • 选择一本书感兴趣的章节优先阅读.
    • 为什么看不懂?
      • 看的不够使劲–书读百遍其义自现.有些时候在阅读的时候会主管的忽略一些点,而器重一些点.
      • 其中有不懂的概念,需要技术性的去理解.去搜索去查阅.
      • 作者讲述的顺序不对，你接着往下看，也许看到后面就明白了前面的了。
  • 在选书的时候如何获取一本书的大致评价
    • 看作者:牛逼的作者写的不会差.
    • 看目录和简介,目录清晰可以透露很多的信息.
    • 看amazon的评价和豆瓣上的评价(着重看低分的评价.)
  • 如何搜索一本好书
    • 同作者的著作
    • amazon相关推荐
    • 一本好的著作里面重点提到的其他著作.
    • 有时对于一个主题,可以搜索到好心人总结的参考资料或者书单.
  • 书写的好处
    • 书写对思维的备忘.
    • 书写是对思维的缓存.
    • 书写是与自己的对话
    • 书写是与别人的交流.:每个人的思维都有一些盲点.
    • 用博客的形式记录下你有价值的思考,会带来很多的好处,却没有任何的明显的坏处
      • 能够交到很多志同道合的朋友.
        • “教”是最好的”学”: 输出.
        • 讨论是绝佳的反思
        • 激励自己去继续学习和思考
        • 学会持之以恒地去做一件事情
        • 一个长期的价值博客是一份很好的简历
        • 让自己成为一个持续学习和思考的人,并只写自己思考和总结之后的产物,其他一切就会随之而来.

遇到问题为什么应该自己动手

* 遇到问题寻找捷径是很聪明的做法.(cost/return分析)
    * 在学习新东西的时候,我们总是想放弃.尤其遇到分水岭的时候.
* 遇到问题寻找捷径为什么只是小聪明
    * 用最小的代价解决手头的问题,看上去寻找经济上最优解的法子.
    * 局部最优or全局最优,取决于你是通过什么方法解决的,
    * 苦难的路越走越容易,容易的路越走越难.

什么才是你的不可代替和核心竞争力.

* 知识技能的组合是最具有相当成都的不可代替性的.
* 专业领域技能:专业性越强,在专业领域的不可替代性就越高.
* 跨领域的技能:解决问题的能力,创新思维,判断与决策能力, 表达沟通能力
* 学习能力:持续学习和思考新的知识.
* 性格要素: 重要的性格:专注,持之以恒,自省,好奇心,自信,谦卑.



    * 碎碎念,心情锁记,唠唠叨叨,没话找话都不算思考.
