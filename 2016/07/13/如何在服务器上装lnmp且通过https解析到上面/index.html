如何在服务器上装lnmp且通过https解析到上面

发表于 2016-07-13   |   分类于 linux   |     |  

服务器安装lnmp

提前准备

  • 一台电脑 —我的是mac系统
  • 一个域名—-在阿里云上,godaddy,namecheap 他们各个的不同,就不介绍了
  • 一台linux服务器–国内阿里云,我用的是vultr,可搭梯子,10美元6个月,最便宜的那种.

连接服务器

  • ssh root@110.110.110.110
  • 然后输入密码.

安装lnmp—linux+nginx+mysql+php

  • 一个个的配置,太烦了,就直接用网上的lnmp搭配lnmp一键安装包
  • 大约半小时你就安装好了.这个是lnmp伪静态管理
  • 这个是lnmp文件所在的位置lnmp所在的位置
  • 这个是简单的lnmp命令
  • 安装好以后通过ip是可以访问到服务器的.

我的配置

wget -c http://soft.vpser.net/lnmp/lnmp1.3-full.tar.gz && tar zxf lnmp1.3-full.tar.gz && cd lnmp1.3-full && ./install.sh lnmp

  • 输入服务器的密码
    之后都是跟这lnmp安装来

域名解析

  • 我的服务器ip是104.156.238.93
  • 域名在阿里云上这样解析就能通过域名访问服务器了.

通过https加密

  • 因为现在域名是通过默认的访问的需要加个虚拟主机

  • 之后一路按回车.

  • 到html目录中.cd /home/wwwroot/www.dong99.com/

  • 新建一个文件 里面写点东西然后访问www.dong99.com 是否出现写的东西vim index.html

  • 到home文件夹 cd /home

  • 新建文件夹ssl mkdir ssl

  • 下载Let’s encrypt 证书的脚本 到ssl 文件夹中

    • 1.wget https://raw.githubusercontent.com/xdtianyu/scripts/master/lets-encrypt/letsencrypt.conf
    • 2.wget https://raw.githubusercontent.com/xdtianyu/scripts/master/lets-encrypt/letsencrypt.sh
    • 3.更改执行权限chmod +x letsencrypt.sh
    • 查询文件有几个 ls 总共有2个文件 一个letsencrpt.sh 一个是letsencrypt.conf

  • 更改letsencrypt.conf文件 vim letsencrypt.conf
    更改内容,第一个是原内容,第二个是我修改后内容

  • 修改完之后运行./letsencrypt.sh letsencrypt.conf

    • 出现一个这样问题

    • 是英文在nginx 中是不能允许在浏览器上访问隐藏文件夹,所以需要去修改nginx的配置.

    • 然后重新运行./letsencrypt.sh letsencrypt.conf


      这个表示成功了.

    • ls 看下文件有几个


      里面有9个文件,其中www.crt-OLD-160713-150415是可以删除了,这个是刚刚运行错误生成的文件.

修改nignx配置

  • 到vhost文件夹cd /usr/local/nginx/conf/vhost
  • 编辑文件 vim www.dong99.com.conf

  • 重定向到https 301

打完收工.但是letsencrypt 证书只能90天,我们需要写个定时的任务每月执行一次.

  1. 进入cron定时任务编辑crontab -e
  2. 输入0 0 1 * * /home/ssl/letsencrypt.sh /home/ssl/letsencrypt.conf >> /var/log/lets-encrypt.log 2>&1
    关于这个定时任务是每个月自动执行 letsencrypt.sh letsencrypt.conf 并且追加到log文件中,，2>&1，表示2的错误输出重定向等同于1.
  3. 查询这定时任务crontab -l
  4. 到ssl文件夹中编辑letsencrypt.sh加入每次更新完重启lnmp功能cd /home/ssl/letsencrypt.sh
  5. 到最后一行 将# 去掉,将最后一行改为lnmp restart.如果你是其他重启命名就改为其他.
